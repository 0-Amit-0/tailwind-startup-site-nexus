# NovaLaunch - Startup Landing Page 🚀

A modern, responsive landing page designed for startups using **Tailwind CSS** and **Vite**. Features a sleek dark theme with smooth animations and an optimized user experience.

---

## 🌟 Features

- **Modern Dark Theme** - Professional dark UI with cyan & indigo accents
- **Fully Responsive** - Adapts seamlessly to mobile, tablet, and desktop screens
- **Smooth Animations** - Blur effects, gradient overlays, and hover transitions
- **Performance Optimized** - Built with Vite for fast loading times
- **Interactive Dashboard** - Real-time statistics visualization
- **SEO Friendly** - Semantic HTML structure with proper metadata
- **Accessibility Ready** - Clean navigation and proper heading hierarchy

---

## 🎨 Design Highlights

- **Hero Section** - Eye-catching headline with animated gradient background
- **Feature Cards** - Organized into Planning, Execution, and Insights sections
- **Statistics Panel** - Shows key metrics: tasks automated, time saved, team sync
- **Call-to-Action** - Multiple engagement points (Free Trial, Demo booking)
- **Sticky Navigation** - Easy access to features and contact sections
- **Backdrop Blur Effects** - Modern glass-morphism styling

## 🚀 Getting Started

### Prerequisites
- Node.js (v14 or higher)
- npm or yarn

### Installation

1. **Clone the repository**
```bash
git clone https://github.com/0-Amit-0/tailwind-startup-site-nexus.git
cd tailwind-startup-site-nexus
```

2. **Install dependencies**
```bash
npm install
```

3. **Run development server**
```bash
npm run dev
```
The site will be available at `http://localhost:5173`

4. **Build for production**
```bash
npm run build
```

## 🎨 Customization

### Update Brand Colors
Edit `src/style.css` or modify Tailwind classes:
- Primary: `cyan-300` / `cyan-400`
- Secondary: `indigo-300` / `indigo-400`
- Dark Background: `slate-950` / `slate-900`

### Change Content
- Hero title & description: Edit `index.html` lines 33-39
- Features: Update cards in lines 94-110
- Statistics: Modify metrics in lines 62-82

### Add More Sections
Use the existing card components as templates and extend with new sections.
